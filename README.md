# Problem Breakdown:
## Algorithm Name: Playlist Vibe Builder using Merge Sort

Merge sort works for this problem because it will break the list of songs into smaller parts, sort the smaller parts based on the key the user chooses, and finally merge the sorted lists together. It's hard to pick a pivot value for quick sort because each song has different properties that make it hard to find a song that is in the middle.

## Problem Breakdown & Computational Thinking

### Decomposition:
1) Create a list of songs with their details (title, duration, chart ranking etc.)
2) Ask user to choose a key for sorting
3) check if the key chosen is an option for sorting
4) use merge sort to sort the songs according to the key
5) output the sorted list of songs to the user

### Abstraction:
- details like the duration, title, ranking
- 3 </= # of songs </= 15

### Pattern Recognition:
- the sorting will have basically the same process, the only thing that really changes is the key

### Algorithmic Thinking:
Input: key

Output: sorted list of songs

Restrictions:
- 3 </= # of songs </= 15
- user needs to enter a valid key (not in the options of keys, not a valid option)
